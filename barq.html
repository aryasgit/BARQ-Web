About This Project
This project implements a fully autonomous quadruped robot controller in Python. The entire stack runs on the robot — no offboard compute, no ROS — covering 3D inverse kinematics, gait generation, IMU-based posture stabilisation, joint-space normalisation, hardware abstraction, tricks, and full gamepad control.

The meaning of this build: every layer from sensor to actuator, understood and owned. No black boxes. No middleware magic. Pure Python from IMU to servo.

The stack is divided into numbered layers. Each layer has exactly one responsibility and communicates only with adjacent layers. You can run any layer in isolation — import the IK solver in a notebook, test posture on a bench, swap the gait generator without touching hardware.

Software Architecture

Layer by Layer,
Top to Bottom.

L6
Gait Generator gait/generator.py
Produces foot targets over time — trot gait with configurable step length, height, frequency
L5
Posture Controller joints/posture.py
IMU feedback → foot target compensation. Roll + pitch stabilisation via Cartesian foot offsets
L3
IK Solver ik/solver.py
Hip-local (x,y,z) → joint deltas. Exact geometric IK for SpotMicro leg geometry
L2.5
Joint Conventions joints/conventions.py
Sign corrections + zero offsets. Resolves servo mount mirror asymmetry between left and right legs
L2
Joint Space joints/space.py
IK deltas → physical servo angles (0-270 deg). Math world ends here, hardware begins
L1
Hardware Drivers hardware/*.py
PCA9685 PWM driver, MPU6050 IMU + complementary filter, shared I2C bus singleton
L0
Absolute Truths hardware/absolute_truths.py
ALL physical constants — channel maps, servo limits, stand angles. Single source of truth for the entire stack
